Rails Asset Pipeline

The rails asset pipline with sprockets is very convienent for quickly building small apps or prototyping.  The asset pipelie has many advantages like handling all processing without having to use a build tool.  As apps begin to grow in size the asset pipeline begins to show some disadvanages like:
  • Delivering all assets to every page load even if the asset is not required for the page.
  • Lost CSS that you may not even be using in your app is included.
  • Keeping dependencies in the HEAD and not in the code.
  • Non Modular approach.
  • Easy to lose control of load order.
  • Difficulty building a truly seperated client.
  • All css and javascript gets minified into a single file.

Webpack


Webpack takes a different approach than the asset pipeline as it is a module bundler.

Pros:

  • Code splitting
  • CLI
  • Dev Server
  • Hot Reloading
  • ES6 module support
  • Supports CommonJS (NPM packages) and AMD
  • Manages JS, CSS, and image assets
  • Many plugins available
  • Shared client and server modules

Cons:

  • Complicated setup
  • Confusing configuration
  • Different approach than sprockets makes it difficult to use side by side

React On Rails

To use React with Rails there are a few options each comes with itâ€™s own set of Proâ€™s and Con's

react-rails gem

pros:

  1. Easty to install and start using right away.
  2. Officially supported by the react team.
  3. Build in SSR support.

cons:

  1. Tied to the asset pipeline.
  2. Support for additional libraries (react-router) is limited to existing gems.
  3. Browser back button can cause some uninted behavior.

Seperate Rails and React Apps

pros:

  1. You can build a full client side application and leverage the rails API.
  2. Each framework does what it does best!

cons:

  1. 2 codebases to maintain.
  2. Double the hosting cost.
For me these were the two best options for combining Rails and React.  Both had obvious limitations and usually the cons outweighed the pros as the application grew.  I began to research how to integrate webpack with Rails.  All of the solutions that I found completely removed the asset pipeline.  I wanted a solution that fulfilled the following requirments:
  1. Webpack working along side sprockets (not one or the other)
  2. A full React App living inside of the Rails codebase.
  3. Layout capabilities for shared assets.
  4. Layout omission for multiple entry points.
After searching for awhile I found the webpack-rails gem.  This Gem solved many of my problems but I also wanted some boilerplate setup for common React, Redux, and react-router integrations.  Out of this the repack gem was born.

repack

This gem incorpeates webpack into rails and sets up a react client for you.  The gem also comes with 2 flags that can be used together, 1 at a time, or not at all. (â€”redux, â€”router)

First we will build out our App container

client/containers/App.js
import React from 'react';
import AddTodo from '../components/AddTodo';
import TodoList from '../components/TodoList';
class App extends React.Component {
constructor(props) {
super(props);
this.state = { todos: [] };
}
componentDidMount() {
$.ajax({
url: '/api/items',
type: 'GET',
dataType: 'JSON'
}).done( todos => {
this.setState({ todos })
})
}
updateList(name) {
$.ajax({
url: '/api/items',
type: 'POST',
dataType: 'JSON',
data: { item: { name }}
}).done( result => {
this.setState({
todos: [
{ ...result },
...this.state.todos
]
})
})
}
updateTodo(id) {
$.ajax({
url: `/api/items/${id}`,
type: 'PUT',
dataType: 'JSON'
}).done( item => {
let index = this.state.todos.findIndex( i => i.id === item.id )
let items = this.state.todos
this.setState({
todos: [
...items.slice(0, index),
{...items[index], complete: item.complete},
...items.slice(index + 1, items.length)
]
})
})
}
deleteTodo(id) {
$.ajax({
url: `/api/items/${id}`,
type: 'DELETE'
}).done( () => {
let index = this.state.todos.findIndex( i => i.id === id )
let items = this.state.todos
this.setState({
todos: [
...items.slice(0, index),
...items.slice(index + 1, items.length)
]
})
})
}
render() {
return (
<div className="container">
<AddTodo updateList={this.updateList.bind(this)} />
<TodoList
todos={this.state.todos}
updateTodo={this.updateTodo.bind(this)}
deleteTodo={this.deleteTodo.bind(this)}
/>
</div>
)
}
}
export default App;
